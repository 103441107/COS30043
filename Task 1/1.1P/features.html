Features implemented on all 3 HTML pages.

About the implemented features

Below are the list of features implemented in all html pages using javascript and css.

  1. In index.html Webpage layout, uses Responsive.
  2. In index.html Colour contrast, uses Mixed.
  3. In index.html Main colours used, uses Black, Light Grey, Orange and White.
  4. In CSS Styles of index.html webpage, uses Bootstrap Library.
  5. In the navigation bar of all html webpages, the link in the nav bar is highlighted when user is on that current page.
  6. In the navigation bar of all html webpages, the animation in the nav bar is Underline From Left.
  7. In index.html, uses Fade In animation.
  8. In index.html #Our Menu, uses flaticon.
  9. In index.html #Types of Pizza, uses Wikipedia library.
  10. In index.html #Our Location, uses Google Map iframe.
  11. All fields in Register.html and Order.html are required to be filled in, if not all fields or there are missing fields not filled in, an error message will show.
  12. In register.html, if no is selected in "Would you like to tell us about your favorite pizza, the div with id="favpizza" will be hidden, and users can proceed to submit form without receiving error message from the fields inside that div.
  13. In register.html, if Email and Re-type email are not the same, error message will be shown upon submit. This also applies to password and re-type password. In addition, password and re-type password must be 8 characters long.
  14. In order.html, Pick up is the default option in the initial load of page in which automatically hides the billing address, and same as delivery checkbox for billing address. In addition, online is also the default option for 'payment method which shows the online method for credit card information.
  15. In order.html, if users selects delivery method, the div for delivery address fields shows up as well as the same as delivery address. In addition if delivery method is selected, the payment method will be hidden. Thus, no error message will be received from the payment method if delivery method is selected.
  16. In order.html, if 'same as delivery address is ticked after users filled in their delivery address, billing address will be automatically filled in. If 'same as delivery address is ticked before filling in the fields in delivery address, error message will be shown telling users to fill in delivery address first.
  17. In order.html, postcode must be 4 digit long, and cvv/ credit card pin must be 3 digit long.
  18. In order.html, if users select their credit card type, credit card number must be of a specific numbered of digits entered. Such as , if users select visa or master card, the credit card number must be 16 digits long, and if users select american express, their credit card number must be 15 digits long.
